# K_League-analysis
K-League 데이터 분석 프로젝트 기술 분석 보고서
본 프로젝트는 축구 경기 중 발생하는 액션 데이터를 바탕으로, 다음 액션이 발생할 위치(x,y 좌표)를 예측하는 머신러닝 시스템입니다.

1. 프로젝트 아키텍처 개요
이 프로젝트는 학습(Train) → 저장(Serialization) → 추론(Inference)의 전 과정을 모듈화하여 설계되었습니다. 
특히 하나의 방식에 안주하지 않고, 좌표를 해석하는 관점을 달리한 두 가지 전략(Tactics vs Plan B)을 병행한 것이 특징입니다.

Tactics (기본 전략): 일반적인 Cartesian 좌표계(x,y) 기반의 예측.

Plan B (심화 전략): 기하학적 관점의 극좌표계(Polar Coordinates) 기반의 예측.


2. 데이터 전처리 및 피처 엔지니어링 (The Brain)
성능을 결정지은 핵심 로직은 train_tactics.py와 train_planb.py의 피처 생성 부분에 있습니다.

2.1 시계열 및 상대적 피처 생성

축구는 이전 상황이 다음 상황에 영향을 미치는 시계열 데이터입니다. 이를 위해 shift(1)를 활용해 '직전 액션'의 정보를 현재 행에 붙였습니다.

prev_end_x, prev_end_y: 직전 액션이 끝난 지점이 현재 액션의 시작점에 미치는 영향 분석.

dist_prev, angle_prev: 이전 지점으로부터 얼마나 멀리, 어떤 각도로 이동했는지 계산.

speed: 시간 차이(prev_time)와 이동 거리를 이용해 액션의 속도감 반영.

2.2 도메인 특화 피처 (Goal-oriented)

축구의 목적은 '골'입니다. 이를 데이터에 반영하기 위해 골대(105,34)를 기준으로 한 피처를 생성했습니다.

dist_goal: 현재 위치에서 골대까지의 직선거리.

angle_goal: 골대를 바라보는 각도.

2.3 공간의 의미론적 해석 (K-Means Clustering)

단순한 좌표값 대신, 경기장을 특정 구역으로 나누어 모델이 '공간적 의미'를 학습하게 했습니다.

K-Means 알고리즘: 경기장 좌표를 클러스터링하여 zone_id 생성. 이를 통해 '박스 안', '측면', '미드필드' 등의 공간 정보를 모델에 주입했습니다.


3. 전략적 실험: Tactics vs Plan B
가장 인상적인 부분은 타겟 변수를 정의하는 방식의 변화입니다.

🔹 Tactics: 직교 좌표계 (Cartesian)

Target: diff_x, diff_y (이동 변화량)

특징: 구현이 직관적이며 일반적인 움직임 패턴을 잡기에 유리함.

🔹 Plan B: 극좌표계 (Polar Coordinates)

Target: target_dist (거리), target_angle (각도)

특징: 축구의 패스나 슈팅은 특정 지점을 향한 '방향'과 '강도(거리)'로 결정된다는 점에 착안.

변환 공식: 

$$Distance = \sqrt{\Delta x^2 + \Delta y^2}$$
Angle=arctan2(Δy,Δx)


4. 모델 아키텍처 (Ensemble & Multi-output)
학습 코드(train_...py)에는 고도화된 모델링 기법이 적용되었습니다.

4.1 앙상블 학습 (Voting Regressor)

단일 모델의 오버피팅을 방지하고 일반화 성능을 높이기 위해, 서로 다른 설정을 가진 HistGradientBoostingRegressor 3개를 하나로 묶었습니다.

model1 (안정성), model2 (복잡도), model3 (균형) 세 모델의 예측값을 평균 내어 최종 결과를 도출합니다.

4.2 다중 출력 회귀 (MultiOutput Regressor)

좌표 예측은 x와 y (혹은 거리와 각도) 두 개의 값을 동시에 예측해야 합니다. 이를 위해 MultiOutputRegressor 래퍼를 사용하여 독립적이면서도 유기적인 다중 타겟 예측을 수행했습니다.


5. 추론 및 모델 관리 파이프라인
predict_...py 파일들은 실제 서비스 환경에서도 돌아갈 수 있을 만큼 체계적입니다.

모델 직렬화 (Serialization): 학습된 모델뿐만 아니라, 범주형 데이터를 숫자로 바꾼 LabelEncoder, 공간을 나눈 KMeans 객체까지 모두 .pkl 파일로 저장하여 추론 시 동일한 환경을 재현했습니다.

타겟 인코딩 (Target Encoding) 통계 활용: 선수별(player_id), 팀별(team_id) 평균 이동 거리 등을 별도의 .csv 통계표로 만들어, 모델이 개별 주체의 성향을 반영할 수 있도록 설계했습니다.

추론 시 복원: Plan B 모델의 경우, 예측된 (거리, 각도)를 다시 (x,y) 좌표로 복원하여 최종 제출 형식을 맞추는 후처리 로직이 포함되어 있습니다.


6. 핵심 인사이트 요약
이 프로젝트는 단순히 등수를 올리기 위한 코딩이 아니라, 데이터 사이언스의 핵심 프로세스를 정석대로 밟고 있습니다.

시계열적 관점: 과거 데이터의 전이 학습.

도메인적 관점: 축구 경기장의 지리적 특성 반영.

기하학적 관점: 좌표계 변환을 통한 새로운 피처 공간 탐색.

엔지니어링적 관점: 모델 모듈화 및 직렬화를 통한 재사용성 확보.
